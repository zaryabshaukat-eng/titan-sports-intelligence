# TITAN — Sports Intelligence OS

Enterprise-grade sports intelligence operating system combining market data, statistics, and explainable AI.

## Stack

- **Runtime:** Bun
- **Framework:** TanStack Start (SSR) + React 19
- **Styling:** Tailwind CSS v4 + custom design tokens (oklch colour space)
- **Charts:** Recharts
- **Routing:** TanStack Router (file-based, auto-generated routeTree)
- **Resizable panels:** react-resizable-panels v4.x (`Group`, `Panel`, `Separator`)
- **Config:** `@lovable.dev/vite-tanstack-config` wraps Vite + Nitro

## How to run

```
bun run dev   # starts the Vite dev server on port 5000
```

The workflow **Start application** is pre-configured.

## Key implementation notes

- **react-resizable-panels v4.x** uses `Group`/`Panel`/`Separator` (not `PanelGroup`/`PanelResizeHandle`) and `orientation` not `direction`.
- **TanStack Router code-splitter**: module-level exported consts from sibling files can be `undefined` inside `?tsr-split=component`. Fix: drop the prop and let the component use its own internal default.
- **StatusPill / status maps**: always use `?? fallback` when indexing `Record<Status, …>` — never assume incoming values are valid keys.
- **Theme system**: applied via `data-theme` attribute on `<html>` element. `ThemeProvider` at `src/components/titan/ThemeProvider.tsx`; persisted to `localStorage` key `titan_theme`. Default (Titan Dark) uses empty string for the attribute.
- The hydration mismatch warning from TanStack devtools (`data-tsd-source`) is cosmetic and harmless.
- All data is placeholder/mock — no external API connections are required to run.

## Features

### Batch 1 — Core enhancements
1. **Global Command Palette** (Ctrl+K) — categorised commands, fuzzy search
2. **Notification Center** — sliding panel, tabs, mark-read
3. **Live Activity Center** — real-time metrics panel
4. **Research Workspace** (`/research`) — 3-column resizable layout with markdown editor
5. **System Status** (`/system-status`) — infrastructure health, incident history
6. **Enhanced AI Intelligence** — per-engine CPU/memory/latency/uptime, area charts
7. **Confidence Widgets** — ConfidenceGauge, ProbabilityRing, RiskMeter, SignalStrength, MarketStrength, HealthIndicator, AccuracyWidget, TrendIndicator
8. **DataTable** — sortable/filterable/paginated, CSV export, density toggle, row selection
9. **Mobile sidebar** — hamburger nav, persistent collapse state (`localStorage`)
10. **Keyboard Shortcuts** — `?` modal, G-chord navigation (`useKeyboardShortcuts` hook)

### Batch 2 — New routes
11. **Historical Timeline** (`/timeline`) — 7 event types, filter bar, time/type grouping
12. **Report Viewer** (`/reports`) — list + in-page viewer, sticky TOC, print/JSON export
13. **Design System** (`/design-system`) — living component catalog, sticky nav
14. **ds.tsx** — Badge, StatusChip, ProgressBar, Skeleton, EmptyState, Dialog, Drawer, MetricCard, Panel, ChartTooltip

### Batch 3 — Accessibility, Themes, Micro-interactions, Performance
15. **Theme System** — 4 themes: Titan Dark, Midnight, Deep Blue, Carbon; palette picker in topbar; `localStorage` persistence
16. **Accessibility** — skip-nav link (`#main-content`), `aria-label` on every interactive element, `aria-current="page"` on active nav items, `aria-sort` on table columns, `role` / `aria-expanded` / `aria-haspopup` throughout, visible `:focus-visible` ring, `prefers-contrast: more` support
17. **Micro-interactions** — `btn-press`, `card-lift`, `page-enter`, `drawer-enter`, `modal-enter`, `stagger-enter`, `badge-pulse`, `value-update` flash, `table-row-interactive`
18. **Reduced motion** — `@media (prefers-reduced-motion: reduce)` disables all animations globally
19. **Performance** — `Timeline` wrapped in `React.memo`; `DataTable` uses `useMemo`/`useCallback` throughout; `ThemeProvider` applies theme synchronously on mount
